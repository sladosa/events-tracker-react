📋 Overview — Sharing model
RoleCan doCannot do
Owner Everything + Share + Manage access —
Grantee write Add, Edit own events, Export Edit structure, Manage shares, Delete others' events
Grantee read View events, Export Add, Edit, Structure changes
💡
Key principle: Sharing is per-Area. Owner keeps full control of structure. User indicators only appear when an Area is shared (hidden for single-user Areas).
🔑
Identity: Each event records who logged it. Display name comes from profiles.display_name (fallback: email). Users set their own display name in Profile settings.

  • View all events (yours + shared users)
  • Add and edit your own events
  • Edit structure (categories, attributes)
  • Invite users (read or write)
  • Revoke access at any time

📊 Excel Export / Import — collab scenarios

7A — "User" kolona u Excel formatu

Svaki red u Activities sheetu = jedna sesija = uvijek jedan korisnik → jedna email kolona, ne lista.

Primjer Activities Events sheeta sa User kolonom NEW col:

A — Date B — Time C — Category_Path D — User NEW E — attr1 F — attr2 ...
2026-04-0209:15Fitness > Activity usera@test.com Running5.2...
2026-04-0110:30Fitness > Activity userb@test.com Gym60...
2026-03-3108:00Fitness > Activity usera@test.com Yoga45...
💡 Format: email (ne display_name) — stabilan identifikator koji se ne mijenja. Kolona D se ubacuje iza Category_Path, ispred atributa. Za non-shared Areas kolona postoji ali je uvijek isti email — nema vizualne razlike za single-user.
🔑 Pozicija kolone: D (iza Category_Path) je konzistentna s trenutnim formatom gdje A=Date, B=Time, C=Category_Path. Atributi se pomiču za jedan stupac desno — breaking change! Import mora ignorirati User kolonu ako ne zna za nju (stari Excel format).

7B — Export scenariji

Tko exportira, što vidi u Excelu.

Tko exportira Koji eventi u Excelu User kolona Napomena
Owner — vlastita (non-shared) Area Samo vlastiti eventi Da (uvijek isti email) — ili sakrij? Nema promjene vs danas
Owner — shared Area Svi eventi (vlastiti + svih grantee-a) Da — miješani emaili RLS propušta sve; ovo je puni Backup
Grantee write Svi eventi (vlastiti + owner-ovi) Da — miješani emaili RLS propušta sve; isto kao owner export
Grantee read Svi eventi (read-only pristup) Da — miješani emaili Export = Backup za read grantee (odluka D5)
💡 Implementacija: za shared Area, ukloniti .eq('user_id', userId) filter u excelDataLoader.ts kada je sharedContext !== null ili Area ima aktivne shares. RLS automatski filtrira što korisnik smije vidjeti.

7C — Import scenariji i "User" kolona na importu

Ključna odluka: što se radi s User kolonom na importu?

Opcija A — Ignoriraj User kolonu Preporučeno
Import uvijek piše user_id = trenutni korisnik.
  • ✅ Jednostavna implementacija
  • ✅ Konzistentno s P3 (Last non-empty wins)
  • ✅ Grantee može importirati u shared Area
  • ⚠️ Backup/restore: Ana-ini eventi postaju Sašini ako owner reimportira
Warning u ImportModal: "All imported events will be assigned to your account."
Opcija B — Matchaj User po emailu
Import lookup profiles.email → dodijeli event pravom korisniku.
  • ✅ Backup/restore čuva originalne autore
  • ❌ Složena implementacija
  • ❌ Email not found → fallback? Error? Skip?
  • ❌ Owner može "upisivati" evente pod tuđim imenom
Nije za sada — previše edge-caseova.

Konkretni import scenariji:

Scenarij Importira kamo Rezultat (Opcija A) OK?
Owner importira vlastiti backup (vlastita Area) Vlastita Area Svi eventi → owner. User kolona ignorirana. ✅ Idealno
Owner importira full backup (shared, ima Ana-inih) Ista shared Area Ana-ini eventi postaju Owner-ovi (P3: duplicati se skipaju ako identični) ⚠️ Prihvatljivo uz warning
Grantee (write) importira u shared Area Shared Area (owner-ove kategorije) Svi eventi → grantee user_id. RLS dozvoljava. ✅ Korisno
Ana exportira Sašin Area, importira u vlastiti Area Ana-in vlastiti Area (iste kategorije) Svi eventi (i Sašini!) → Ana-in user_id. Vlastita kopija. ✅ Korisno — "uzmi svoju kopiju"
Grantee (read) pokušava importirati Shared Area Blokiran (read grantee ne može INSERT events) ✅ RLS blokira

7D — Breaking change: User kolona mijenja pozicije stupaca

Trenutni format: A=Date, B=Time, C=Category_Path, D=attr1... Nova kolona D=User pomjera atribute.

Stari format (danas)

ABCDE
DateTimeCat_Pathattr1attr2

Novi format (s User kolonom)

ABCDEF
DateTimeCat_PathUserattr1attr2
🔑 Opcija A — Uvijek dodaj User kolonu (i za non-shared Areas): breaking change ali konzistentan format. Import mora handlati oba formata (s i bez User kolone) — detekcija po header vrijednosti.

🔑 Opcija B — User kolona samo za shared Areas: format ostaje kompatibilan za single-user. Import je jednostavniji (ako nema User kolone, sve → current user).

💬 Preporuka: Opcija B — backward compatibility je važna, stari fajlovi moraju raditi.
